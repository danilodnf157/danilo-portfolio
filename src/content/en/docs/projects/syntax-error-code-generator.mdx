    ## From operational pain to standardization

    ### Act 1 — Logs without identity
    In multi‑service environments, errors without clear codes hinder grouping, alerting, and diagnosis — especially as teams grow.

    ### Act 2 — The insight
    Define a **simple, stable convention** (domain prefixes + suffix), generate codes directly in the editor workflow, and use them consistently in every `AppError`.

    ### Act 3 — The proposal
    An extension that:
    - Generates **unique codes** following project standards (e.g., `AUTH_001`, `PAY_402`).  
    - Provides **custom prefixes**, separator, and suffix length.  
    - Supports **contextual analysis** to suggest acronyms (USR, DB, VAL, NET…).  
    - Prevents duplication and speeds up **production debugging**.

    ### Act 4 — Expected impact
    - Better **grouping** and **alerts** in observability tools.  
    - Reduced **diagnosis time** through traceable codes.  
    - Superior **DX** via keyboard shortcuts and editor integration.

    ### Act 5 — Next steps
    Language‑specific formatters, monorepo presets, and `AppError` generator integrations.
  </section>

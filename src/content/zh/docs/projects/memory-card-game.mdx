    ## 从简单想法到完整的 PWA 记忆卡游戏

    ### 第一幕 —— 选择正确的练习
    我希望在不依赖复杂后端的前提下，强化推理与工程实践。记忆卡游戏似乎“过于简单”—— 直到我把它设为 UX、可访问性与 PWA 的综合练兵场。

    <Callout type="warning" title="假设">
      只要足够流畅、可访问并提供清晰的进度（成就/模式），它就能从休闲小游戏升级为“有粘性”的体验。
    </Callout>

    ### 第二幕 —— 设计洞见
    将“翻牌”拆解为一系列微决策：节奏、展示时机、视听提示与难度曲线，目标是在保持流畅的同时避免认知疲劳。

    ### 第三幕 —— 构建
    - **架构：** React + TypeScript，Vite 构建，完整 PWA（可安装 + 离线）。
    - **循环：** 精准的计时、进度持久化与 4 种模式。
    - **可访问性：** 完整键盘与读屏支持、受控焦点与 ARIA。
    - **反馈：** 每个关键事件均配以同步动画与音效。

    ### 第四幕 —— 预期影响
    - 因成就与模式多样而延长会话时长。
    - 通过清晰提示与稳定节奏减少“暴躁退出”。
    - PWA 离线游玩带来高复玩性，尤其适合移动端。

    <Callout type="success" title="实践成果">
      我的第二个网页游戏，将可访问 UX 与前端工程最佳实践应用于原生网页游戏。
    </Callout>

    ### 第五幕 —— 留下的资产
    除了乐趣，还沉淀了一套可复用脚手架，用于快速原型化强调进度、性能与可访问性的网页游戏。
  </section>

  <section className="mt-6">
    ## 模式与成就

    - **经典模式：** 随机配对，标准节奏。
    - **限时挑战：** 倒计时，快速连击奖励。
    - **禅模式（Zen）：** 强调专注与准确，无时间压力。
    - **本地对战：** 两位玩家同一设备对弈。
    - **成就（示例）：** “零失误”“极速玩家”“序列大师”“离线马拉松（PWA）”。

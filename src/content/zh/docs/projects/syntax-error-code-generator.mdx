    ## 从运维痛点到标准化

    ### 第一幕 — 无标识的日志
    在多服务环境下，缺少清晰错误码会阻碍日志分组、告警与诊断，团队规模越大问题越明显。

    ### 第二幕 — 灵感
    约定**简单稳定**的编码规范（领域前缀 + 后缀），在编辑器工作流里直接生成，并在每个 `AppError` 中一致使用。

    ### 第三幕 — 方案
    该扩展：
    - 生成**唯一错误码**（如 `AUTH_001`, `PAY_402`）。  
    - 提供**自定义前缀**、分隔符与后缀长度。  
    - 支持**上下文分析**建议缩写（USR、DB、VAL、NET…）。  
    - 避免重复并加速**生产环境调试**。

    ### 第四幕 — 预期影响
    - 更好的**日志分组**与**告警**。  
    - 通过可追踪的错误码降低**诊断时间**。  
    - 借助快捷键提升 **DX**。

    ### 第五幕 — 下一步
    按语言的格式化器、monorepo 预设以及与 `AppError` 生成器的集成。
